这篇文章整理的是一种基于 `epoll + timerfd + 小顶堆` 的低开销定时方案。

(本文经过叹雪二改，原文请看[酷安原文](https://www.coolapk.com/feed/70492913?s=MjNmZGFhNmExYmMyZTBlZzZhM2UxNDU3egi1631))

这个方案主要是为了解决两类问题：

- 部分场景下，`cron` 定时不够灵活，且周期性唤醒会带来额外开销。
- 直接使用 `sleep` 做长时间等待，容易遇到休眠、漂移、唤醒管理和轮询开销问题。

核心思路很简单：用一个 `timerfd` 表示“最近一次要触发的任务”，用小顶堆维护所有任务的下一次执行时间，再用 `epoll_wait` 阻塞等待事件。这样不管有多少任务，内核侧只需要关注一个定时器文件描述符。

## 方案概览

整体流程可以拆成四步：

1. 把所有定时任务放进小顶堆，堆顶永远是最近要执行的任务。
2. 用堆顶任务的执行时间设置 `timerfd`。
3. 主循环用 `epoll_wait` 阻塞等待 `timerfd` 触发。
4. 任务触发后执行脚本，计算下一次执行时间，再重新入堆。

这样做的好处是：

| 目标 | 实现方式 |
| --- | --- |
| 少占文件描述符 | 多个任务共用一个 `timerfd` |
| 减少 CPU 空转 | `epoll_wait` 阻塞等待，不主动轮询 |
| 快速找到最近任务 | 小顶堆 `O(1)` 取堆顶 |
| 支持大量任务 | 插入和删除任务为 `O(log n)` |

## 为什么不用 sleep

`sleep` 适合非常简单的延迟，但不适合做完整的任务调度器。

如果一个任务每天执行一次，直接 `sleep` 很长时间会带来几个问题：

- 任务变更后不容易立即生效。
- 多个任务需要自己维护复杂的等待逻辑。
- 系统休眠、时间变化、进程暂停都可能影响实际触发时间。
- 如果写成 `sleep + 轮询`，还会带来额外 CPU 开销。

所以，更稳的做法是把“等待”交给内核定时器，把“任务排序”交给用户态数据结构。

## 为什么不用纯 cron

`cron` 的优势是成熟、通用、能做不可休眠定时。但在一些轻量任务调度场景里，它也有明显取舍：

- 通常按分钟粒度触发，不适合更灵活的任务控制。
- 周期性唤醒会带来额外开销。
- 任务数量多时，配置和状态管理不如进程内调度灵活。

如果需求是“亮屏或进程活跃时才需要定时”，`timerfd + epoll` 这种可休眠方案通常更省资源。

## 核心数据结构：小顶堆

多任务定时的关键问题是：如何快速找到最近要执行的任务？

这里采用优先队列，也就是小顶堆。堆顶保存下一次执行时间最早的任务。每次任务变化后，只需要重新取堆顶并设置 `timerfd`。

相比有序链表，小顶堆在任务数量较多时更合适：

| 数据结构 | 获取最近任务 | 插入任务 | 删除任务 |
| --- | --- | --- | --- |
| 有序链表 | `O(1)` | `O(n)` | `O(1)` 或 `O(n)` |
| 小顶堆 | `O(1)` | `O(log n)` | `O(log n)` |

## C++ 任务队列示例

`std::priority_queue` 默认是最大堆。如果要做最小堆，比较函数需要反过来写。

```cpp
/**
 * @brief 任务比较函数对象，用于优先队列排序
 * @details 按照下次执行时间从小到大排序，时间越早优先级越高
 */
struct TaskCompare {
    bool operator()(const ScheduledTask& a, const ScheduledTask& b) const {
        // priority_queue 默认是最大堆，所以用 > 实现最小堆
        return a.next_execution > b.next_execution;
    }
};

using TaskQueue = std::priority_queue<
    ScheduledTask,
    std::vector<ScheduledTask>,
    TaskCompare
>;
```

这个队列会自动把最近要执行的任务放在堆顶：

```cpp
const ScheduledTask& next_task = task_queue.top();
```

int task_compare(const ScheduledTask *a, const ScheduledTask *b) {

typedef struct {

## 最小堆的基本操作

下面是一个简化版最小堆，用整数演示入堆、取堆顶和弹出堆顶。

```c
#include <stdio.h>

#define MAX_SIZE 100

int heap[MAX_SIZE];
int heap_size = 0;

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

// 插入后上浮，维护最小堆
void heap_up(int idx) {
    while (idx > 0) {
        int parent = (idx - 1) / 2;

        if (heap[idx] < heap[parent]) {
            swap(&heap[idx], &heap[parent]);
            idx = parent;
        } else {
            break;
        }
    }
}

// 弹出堆顶后下沉，维护最小堆
void heap_down(int idx) {
    while (1) {
        int left = idx * 2 + 1;
        int right = idx * 2 + 2;
        int min_idx = idx;

        if (left < heap_size && heap[left] < heap[min_idx])
            min_idx = left;

        if (right < heap_size && heap[right] < heap[min_idx])
            min_idx = right;

        if (min_idx != idx) {
            swap(&heap[idx], &heap[min_idx]);
            idx = min_idx;
        } else {
            break;
        }
    }
}

void push(int val) {
    if (heap_size >= MAX_SIZE) {
        printf("堆已满\n");
        return;
    }

    heap[heap_size] = val;
    heap_up(heap_size);
    heap_size++;
}

int top() {
    return heap[0];
}

void pop() {
    if (heap_size <= 0)
        return;

    heap[0] = heap[heap_size - 1];
    heap_size--;
    heap_down(0);
}

int empty() {
    return heap_size == 0;
}

int main() {
    push(30);
    push(10);
    push(50);
    push(20);

    printf("最小堆中的元素:\n");
    while (!empty()) {
        printf("%d\n", top());
        pop();
    }

    return 0;
}
```

输出顺序会是从小到大，说明堆顶始终是最小值。

## 把最近任务映射到 timerfd

有了任务队列后，下一步就是把堆顶任务的执行时间转换成内核定时器。

这里使用 `timerfd_settime`。做法是：

1. 取出堆顶任务。
2. 计算目标执行时间和当前时间的差值。
3. 如果任务已经过期，就把延迟设置为 `0`，让它尽快触发。
4. 调用 `timerfd_settime` 更新定时器。

### C++ 示例

```cpp
// 设置定时器到最近任务的执行时间
inline bool set_timer_to_next_task(int timer_fd, const TaskQueue& task_queue) {
    if (task_queue.empty())
        return false;

    const ScheduledTask& next_task = task_queue.top();
    time_t now = time(nullptr);
    time_t delay = next_task.next_execution - now;

    if (delay < 0)
        delay = 0; // 已过期，立即执行

    struct itimerspec spec = {};
    spec.it_value.tv_sec = delay;

    return timerfd_settime(timer_fd, 0, &spec, nullptr) == 0;
}
```

### C 示例

```c
#include <time.h>
#include <sys/timerfd.h>
#include <stdbool.h>

typedef struct ScheduledTask {
    time_t next_execution;
} ScheduledTask;

typedef struct TaskQueue {
    ScheduledTask* data;
    int size;
} TaskQueue;

static inline bool task_queue_empty(const TaskQueue* q) {
    return q->size <= 0;
}

static inline const ScheduledTask* task_queue_top(const TaskQueue* q) {
    return &q->data[0];
}

static inline bool set_timer_to_next_task(int timer_fd, const TaskQueue* task_queue) {
    if (task_queue_empty(task_queue))
        return false;

    const ScheduledTask* next_task = task_queue_top(task_queue);
    time_t now = time(NULL);
    time_t delay = next_task->next_execution - now;

    if (delay < 0)
        delay = 0; // 已过期，立即执行

    struct itimerspec spec = {0};
    spec.it_value.tv_sec = delay;

    return timerfd_settime(timer_fd, 0, &spec, NULL) == 0;
}
```

## epoll 在这里负责什么

`timerfd` 本质上是一个文件描述符，因此可以直接加入 `epoll`。

主循环可以这样理解：

```text
任务入堆
  ↓
堆顶时间写入 timerfd
  ↓
epoll_wait 阻塞等待
  ↓
timerfd 可读
  ↓
执行到期任务
  ↓
计算下一次执行时间并重新入堆
```

这样就不用自己写轮询，也不需要为每个任务创建一个定时器文件描述符。

## 定时源的选择

`timerfd_create` 需要选择时钟源。不同定时源的行为不一样，尤其是在系统休眠时。

| 定时源 | 休眠时行为 | 适合场景 |
| --- | --- | --- |
| `CLOCK_MONOTONIC` | 深度休眠时通常暂停计时 | 亮屏、常驻进程、低功耗调度 |
| `CLOCK_BOOTTIME` | 包含系统挂起时间 | 希望休眠时间也计入延迟 |
| `CLOCK_REALTIME_ALARM` | 可用于唤醒设备 | 需要准点唤醒的任务 |

如果任务只需要在设备活跃时工作，优先考虑 `CLOCK_MONOTONIC`，开销更低，也更符合可休眠定时的目标。

如果任务必须在系统休眠后也尽量准点执行，可以考虑不可休眠定时源，例如 `CLOCK_REALTIME_ALARM` 或 `CLOCK_BOOTTIME`。这类方案更接近 `cron` 的使用场景，但也意味着更高的唤醒成本。

## 配置格式设计

一个简单的任务配置可以支持三种格式：

| 格式 | 含义 | 示例 |
| --- | --- | --- |
| `HH:MM /path` | 每天固定时间执行 | `03:30 /data/task.sh` |
| `W HH:MM /path` | 每周固定时间执行，`W=1-7` | `1 03:30 /data/task.sh` |
| `N /path` | 每 N 分钟执行一次 | `30 /data/task.sh` |

解析配置时，只需要把调度部分转换成下一次执行时间，再把任务放入小顶堆。

## C++ 配置解析示例

```cpp
/**
 * @brief 解析一行配置并添加到任务队列
 *
 * 格式说明：
 *  - 每日几点几分："HH:MM /path"
 *  - 每周几几点几分："W HH:MM /path"  (W=1-7)
 *  - 每多少分钟执行："N /path"         (N>0)
 */
bool parse_and_schedule_task(const std::string& line, TaskQueue& task_queue) {
    if (line.empty() || line[0] == '#')
        return false;

    // 找到第一个斜杠，分隔调度部分和脚本路径
    size_t first_slash = line.find('/');
    if (first_slash == std::string::npos)
        return false;

    std::string schedule_part = strtrim(line.substr(0, first_slash));
    std::string script_path = strtrim(line.substr(first_slash));

    if (script_path.empty())
        return false;

    ScheduledTask task;
    task.script_path = script_path;

    // 分支 1：纯数字，表示每 N 分钟执行
    if (std::all_of(schedule_part.begin(), schedule_part.end(), ::isdigit)) {
        int minutes = std::stoi(schedule_part);
        if (minutes > 0) {
            task.schedule_type = "minutely";
            task.minutes_interval = minutes;
            task.next_execution = calculate_minutely_time(minutes, time(nullptr));
            task_queue.push(task);
            return true;
        }
    }

    // 分支 2：周几 + 时分
    size_t space_pos = schedule_part.find(' ');
    if (space_pos != std::string::npos) {
        std::string first_part = strtrim(schedule_part.substr(0, space_pos));
        std::string second_part = strtrim(schedule_part.substr(space_pos + 1));

        if (std::all_of(first_part.begin(), first_part.end(), ::isdigit)) {
            int weekday = std::stoi(first_part);
            if (weekday >= 1 && weekday <= 7) {
                int hour, minute;
                if (parse_time_string(second_part, hour, minute)) {
                    task.schedule_type = "weekly";
                    task.weekday = weekday;
                    task.hour = hour;
                    task.minute = minute;
                    task.next_execution = calculate_weekly_time(
                        weekday,
                        hour,
                        minute,
                        time(nullptr)
                    );
                    task_queue.push(task);
                    return true;
                }
            }
        }
    }

    // 分支 3：纯时分，表示每日执行
    int hour, minute;
    if (!parse_time_string(schedule_part, hour, minute))
        return false;

    task.schedule_type = "daily";
    task.hour = hour;
    task.minute = minute;
    task.next_execution = calculate_daily_time(hour, minute, time(nullptr));
    task_queue.push(task);
    return true;
}
```

## C 配置解析示例

下面是一个简化版 C 写法，重点展示解析思路。

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>
#include <time.h>
#include <stdbool.h>

typedef struct {
    char buf[256];
} String;

typedef struct {
    char schedule_type[16];
    char script_path[256];
    int minutes_interval;
    int weekday;
    int hour;
    int minute;
    time_t next_execution;
} ScheduledTask;

typedef struct TaskQueue TaskQueue;
extern void task_queue_push(TaskQueue* q, const ScheduledTask* t);

static void str_trim(String* s);
static bool str_all_digit(const String* s);
static int str_to_int(const String* s);
static bool parse_time_string(const String* s, int* hour, int* minute);
static time_t calculate_minutely_time(int interval, time_t now);
static time_t calculate_daily_time(int h, int m, time_t now);
static time_t calculate_weekly_time(int w, int h, int m, time_t now);

bool parse_and_schedule_task(const char* line, TaskQueue* task_queue) {
    String line_str;
    strncpy(line_str.buf, line, sizeof(line_str.buf) - 1);
    line_str.buf[sizeof(line_str.buf) - 1] = '\0';

    if (strlen(line_str.buf) == 0 || line_str.buf[0] == '#')
        return false;

    char* slash_ptr = strchr(line_str.buf, '/');
    if (!slash_ptr)
        return false;

    String schedule_part;
    String script_path;

    size_t slash_idx = slash_ptr - line_str.buf;
    strncpy(schedule_part.buf, line_str.buf, slash_idx);
    schedule_part.buf[slash_idx] = '\0';
    str_trim(&schedule_part);

    strncpy(script_path.buf, slash_ptr, sizeof(script_path.buf) - 1);
    script_path.buf[sizeof(script_path.buf) - 1] = '\0';
    str_trim(&script_path);

    if (strlen(script_path.buf) == 0)
        return false;

    ScheduledTask task;
    memset(&task, 0, sizeof(task));
    strcpy(task.script_path, script_path.buf);

    if (str_all_digit(&schedule_part)) {
        int minutes = str_to_int(&schedule_part);
        if (minutes > 0) {
            strcpy(task.schedule_type, "minutely");
            task.minutes_interval = minutes;
            task.next_execution = calculate_minutely_time(minutes, time(NULL));
            task_queue_push(task_queue, &task);
            return true;
        }
    }

    char* space_ptr = strchr(schedule_part.buf, ' ');
    if (space_ptr) {
        String first_part;
        String second_part;

        size_t sp_idx = space_ptr - schedule_part.buf;
        strncpy(first_part.buf, schedule_part.buf, sp_idx);
        first_part.buf[sp_idx] = '\0';
        str_trim(&first_part);

        strcpy(second_part.buf, space_ptr + 1);
        str_trim(&second_part);

        if (str_all_digit(&first_part)) {
            int weekday = str_to_int(&first_part);
            if (weekday >= 1 && weekday <= 7) {
                int h, m;
                if (parse_time_string(&second_part, &h, &m)) {
                    strcpy(task.schedule_type, "weekly");
                    task.weekday = weekday;
                    task.hour = h;
                    task.minute = m;
                    task.next_execution = calculate_weekly_time(weekday, h, m, time(NULL));
                    task_queue_push(task_queue, &task);
                    return true;
                }
            }
        }
    }

    int hour, minute;
    if (!parse_time_string(&schedule_part, &hour, &minute))
        return false;

    strcpy(task.schedule_type, "daily");
    task.hour = hour;
    task.minute = minute;
    task.next_execution = calculate_daily_time(hour, minute, time(NULL));
    task_queue_push(task_queue, &task);
    return true;
}

static void str_trim(String* s) {
    char* start = s->buf;
    while (*start && isspace((unsigned char)*start))
        start++;

    char* end = start + strlen(start) - 1;
    while (end >= start && isspace((unsigned char)*end))
        end--;

    *(end + 1) = '\0';
    memmove(s->buf, start, strlen(start) + 1);
}

static bool str_all_digit(const String* s) {
    for (int i = 0; s->buf[i]; i++) {
        if (!isdigit((unsigned char)s->buf[i]))
            return false;
    }

    return strlen(s->buf) > 0;
}

static int str_to_int(const String* s) {
    return atoi(s->buf);
}
```

## 配置热重载

如果定时任务来自配置文件，可以用 `inotify` 监听配置变化。

建议监听 `IN_CLOSE_WRITE` 事件，因为它表示文件写入已经完成，比单纯监听修改事件更稳。

基本流程：

1. 使用 `inotify_add_watch` 监听配置文件。
2. 配置写入完成后触发 `IN_CLOSE_WRITE`。
3. 重新读取配置文件。
4. 清空旧任务队列。
5. 重新计算所有任务的下一次执行时间并入堆。
6. 更新 `timerfd` 到新的堆顶任务。

这样可以实现配置热重载，不需要重启定时进程。

## 其他 Linux 定时方案

Linux 下常见的定时方案有很多，不同方案适合不同场景：

| 方案 | 特点 | 适合场景 |
| --- | --- | --- |
| `alarm` + 信号 | 实现简单，但信号处理复杂，精度和可维护性一般 | 很简单的单次定时 |
| `select/poll/epoll` 超时参数 | 可和 I/O 等待结合，但多个任务需要额外管理 | 网络程序里的简单超时 |
| POSIX timer | 精度和控制能力更好 | 需要标准定时器接口的场景 |
| `timerfd` | 定时器抽象为文件描述符，可直接接入 `epoll` | 事件驱动程序、后台调度器 |
| 时间轮 | 批量管理大量近似定时任务 | 高并发、允许一定精度损失的调度 |
| 小顶堆 + `timerfd` | 精确管理最近任务，只用一个内核定时器 | 多任务调度、低资源占用 |

`select` 的 timeout 参数也能实现定时等待，但它有两个问题：

- 返回后 timeout 可能被修改，每次调用前都要重新设置。
- 可监控文件描述符数量有限，高并发场景下容易成为瓶颈。

因此，在需要更强扩展性和更低开销的后台调度器里，`epoll + timerfd + 小顶堆` 会更自然。

## 总结

这个定时方案的关键不是“创建很多定时器”，而是只把最近一次到期任务交给内核，其余任务由用户态小顶堆管理。

它的核心优势是：

- 任务数量增加时，不需要增加 `timerfd` 数量。
- `epoll_wait` 阻塞等待，避免无意义轮询。
- 小顶堆能快速找到最近任务。
- 可根据场景选择可休眠或不可休眠定时源。
- 配合 `inotify` 可以实现配置热重载。

如果你的定时任务需要低开销、可扩展、能和事件循环结合，那么这套方案会比简单 `sleep` 更稳，也比直接依赖 `cron` 更灵活。
