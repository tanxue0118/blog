## 怎么使用单timerfd来实现多任务定时？

可以采用将堆顶时间映射为内核定时器，有了任务队列后，下一步是如何将堆顶任务的时间转换为内核定时器。这里使用timerfd_settime系统调用，设置定时器在指定延迟后触发。

计算当前时间与目标执行时间的差值，如果差值为负，则设置为立即触发，采用优先队列（小顶堆）来实现这一需求，小顶堆能够在O(1)时间内获取最近任务，在O(log n)时间内完成任务的插入和删除。相比有序链表的O(n)插入，堆结构在任务数量较多时性能优势明显。

**cpp**
```cpp
/**
 * @brief 任务比较函数对象，用于优先队列排序
 * @details 按照下次执行时间从小到大排序（时间越早优先级越高）
 */
struct TaskCompare {
    bool operator()(const ScheduledTask& a, const ScheduledTask& b) const {
        return a.next_execution > b.next_execution;  // 注意：priority_queue是最大堆，所以用>实现最小堆
    }
};

// 自动按执行时间排序，时间堆插入任务（堆的push）=>实现单timerfd低功耗定时方案
using TaskQueue = std::priority_queue<ScheduledTask, std::vector<ScheduledTask>, TaskCompare>;
```

采用小顶堆的好处就是不管多少任务，只占用一个timerfd文件描述符，带来更低的资源占用，epoll采用epoll_wait阻塞等待，无需轮询带来的额外开销。常规cron定时其实并不是最优方案，为什么大部分都选择cron?那就是他是不可休眠定时方案，不会被系统休眠所影响，但是开销其实是比较大的，由于每分钟唤醒机制。采用了优先队列（PriorityQueue）作为核心数据结构，其底层实现是二叉堆。

## 自定义优先级
如果你需要一个最小堆，可以通过自定义比较函数来实现：

**cpp**
```cpp
#include <iostream>
#include <queue>
#include <vector>

struct compare {
    bool operator()(int a, int b) {
        return a > b; // 定义最小堆
    }
};

int main() {
    // 创建一个自定义类型的优先队列，使用最小堆
    std::priority_queue<int, std::vector<int>, compare> pq_min;

    // 向优先队列中添加元素
    pq_min.push(30);
    pq_min.push(10);
    pq_min.push(50);
    pq_min.push(20);

    // 输出队列中的元素
    std::cout << "最小堆中的元素: " << std::endl;
    while (!pq_min.empty()) {
        std::cout << pq_min.top() << std::endl;
        pq_min.pop();
    }

    return 0;
}
```

**c**
```c
#include <stdio.h>

#define MAX_SIZE 100
int heap[MAX_SIZE];
int heap_size = 0;

void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

// 插入上浮，维护最小堆
void heap_up(int idx) {
    while (idx > 0) {
        int parent = (idx - 1) / 2;
        if (heap[idx] < heap[parent]) {
            swap(&heap[idx], &heap[parent]);
            idx = parent;
        } else {
            break;
        }
    }
}

// 删除堆顶下沉，维护最小堆
void heap_down(int idx) {
    while (1) {
        int left = idx * 2 + 1;
        int right = idx * 2 + 2;
        int min_idx = idx;

        if (left < heap_size && heap[left] < heap[min_idx])
            min_idx = left;
        if (right < heap_size && heap[right] < heap[min_idx])
            min_idx = right;

        if (min_idx != idx) {
            swap(&heap[idx], &heap[min_idx]);
            idx = min_idx;
        } else {
            break;
        }
    }
}

// 入堆
void push(int val) {
    if (heap_size >= MAX_SIZE) {
        printf("堆已满\n");
        return;
    }
    heap[heap_size] = val;
    heap_up(heap_size);
    heap_size++;
}

// 获取堆顶
int top() {
    return heap[0];
}

// 弹出堆顶
void pop() {
    if (heap_size <= 0) return;
    heap[0] = heap[heap_size - 1];
    heap_size--;
    heap_down(0);
}

// 判断堆是否为空
int empty() {
    return heap_size == 0;
}

int main() {
    push(30);
    push(10);
    push(50);
    push(20);

    printf("最小堆中的元素:\n");
    while (!empty()) {
        printf("%d\n", top());
        pop();
    }
    return 0;
}
```

为什么不采用传统sleep方案，因为这个在sleep比如时间过长，比如一天一次的任务sleep 24800就会出现定时出问题，达不到cron的不可休眠定时稳定，而且sleep+轮询导致cpu无限空转，带来很大的额外开销。

**差值为负对应到期执行**
**cpp**

```cpp
//设置定时器到最近任务的执行时间, timer_fd 定时器文件描述符, task_queue 任务队列
inline bool set_timer_to_next_task(int timer_fd, const TaskQueue& task_queue) {
    if (task_queue.empty())
        return false;

    const ScheduledTask& next_task = task_queue.top();
    time_t now = time(nullptr);
    time_t delay = next_task.next_execution - now;

    if (delay < 0)
        delay = 0; // 过期执行

    struct itimerspec spec = {};
    spec.it_value.tv_sec = delay;

    return timerfd_settime(timer_fd, 0, &spec, nullptr) == 0;
}
```

```c
typedef struct ScheduledTask {
    time_t next_execution;

    int size;

    return q->size <= 0;

    return &q->data[0];
}

static inline bool set_timer_to_next_task(int timer_fd, const TaskQueue* task_queue) {
    if (task_queue_empty(task_queue))
        return false;

    const ScheduledTask* next_task = task_queue_top(task_queue);
    time_t now = time(NULL);
    time_t delay = next_task->next_execution - now;

    if (delay < 0)
        delay = 0; // 过期执行

    struct itimerspec spec = {0};
    spec.it_value.tv_sec = delay;

    return timerfd_settime(timer_fd, 0, &spec, NULL) == 0;
}
```

## 定时源的选择

针对定时源的选择先说结论可休眠timerfd定时源选择CLOCK_MONOTONIC开销最小，不可休眠定时源采用CLOCK_REALTIME_ALARM和CLOCK_BOOTTIME都行。可休眠定时源的意思是在系统进入深度doze状态定时会暂停。直到退出，定时才继续执行。

**定时源分为可休眠定时和不可休眠定时**

可休眠定时可以用于某常驻进程你又不想系统进入深度doze带来的额外任务开销。比如调度，游戏触控率，一切亮屏才需要的功能都建议采用可休眠定时源，这样能比常规定时方案在休眠时带来更低开销和资源占用。

不可休眠定时源比如常见的cron定时，在系统进入深度doze也能定时准点执行某个任务的方案。

## 定时任务配置解析函数
**cpp**
```cpp
/**
 * @brief 解析一行配置并添加到任务队列
 * @param task_queue 任务队列
 * 格式说明：
 *  - 每日几点几分："HH:MM /path"
 *  - 每周几几点几分："W HH:MM /path"  (W=1-7)
 *  - 每多少分钟执行："N /path"     (N>0)
 */
bool parse_and_schedule_task(const std::string& line, TaskQueue& task_queue) {
    // 忽略空行和注释
    if (line.empty() || line[0] == '#')
        return false;

    // 找到第一个斜杠，分隔调度部分和脚本路径
    size_t first_slash = line.find('/');
    if (first_slash == std::string::npos)
        return false;

    std::string schedule_part = strtrim(line.substr(0, first_slash));
    std::string script_path = strtrim(line.substr(first_slash));

    if (script_path.empty())
        return false;

    ScheduledTask task;
    task.script_path = script_path;

    // 分支1：纯数字 → 每N分钟执行
    if (std::all_of(schedule_part.begin(), schedule_part.end(), ::isdigit)) {
        int minutes = std::stoi(schedule_part);
        if (minutes > 0) {
            task.schedule_type = "minutely";
            task.minutes_interval = minutes;
            task.next_execution = calculate_minutely_time(minutes, time(nullptr));
            task_queue.push(task);
            return true;
        }
    }

    // 分割空格，判断是否为 周数+时分 格式
    size_t space_pos = schedule_part.find(' ');
    if (space_pos != std::string::npos) {
        std::string first_part = strtrim(schedule_part.substr(0, space_pos));
        std::string second_part = strtrim(schedule_part.substr(space_pos + 1));

        // 第一部分是数字，代表周几
        if (std::all_of(first_part.begin(), first_part.end(), ::isdigit)) {
            int weekday = std::stoi(first_part);
            if (weekday >= 1 && weekday <= 7) {
                int hour, minute;
                if (parse_time_string(second_part, hour, minute)) {
                    task.schedule_type = "weekly";
                    task.weekday = weekday;
                    task.hour = hour;
                    task.minute = minute;
                    task.next_execution = calculate_weekly_time(weekday, hour, minute, time(nullptr));
                    task_queue.push(task);
                    return true;
                }
            }
        }
    }

    // 分支3：纯时分 → 每日执行
    int hour, minute;
    if (!parse_time_string(schedule_part, hour, minute))
        return false;

    task.schedule_type = "daily";
    task.hour = hour;
    task.minute = minute;
    task.next_execution = calculate_daily_time(hour, minute, time(nullptr));
    task_queue.push(task);
    return true;
}
```

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <ctype.h>

#include <stdbool.h>

// 简易字符串封装，替代std::string
typedef struct {
    char buf[256];
} String;

// 定时任务结构体
typedef struct {
    char schedule_type[16];
    char script_path[256];
    int minutes_interval;
    int weekday;
    int hour;
    int minute;

} ScheduledTask;

typedef struct TaskQueue TaskQueue;
extern bool task_queue_empty(const TaskQueue* q);
extern const ScheduledTask* task_queue_top(const TaskQueue* q);
extern void task_queue_push(TaskQueue* q, const ScheduledTask* t);

static int str_to_int(const String* s);

static time_t calculate_minutely_time(int interval, time_t now);
static time_t calculate_daily_time(int h, int m, time_t now);
static time_t calculate_weekly_time(int w, int h, int m, time_t now);

// 解析一行配置并添加到任务队列
bool parse_and_schedule_task(const char* line, TaskQueue* task_queue) {
    String line_str;
    strncpy(line_str.buf, line, sizeof(line_str.buf)-1);
    line_str.buf[sizeof(line_str.buf)-1] = '\0';

    // 空行 / 注释跳过
    if (strlen(line_str.buf) == 0 || line_str.buf[0] == '#')
        return false;

    // 查找第一个 '/'
    char* slash_ptr = strchr(line_str.buf, '/');
    if (!slash_ptr)
        return false;

    String schedule_part, script_path;
    // 分割调度部分
    size_t slash_idx = slash_ptr - line_str.buf;
    strncpy(schedule_part.buf, line_str.buf, slash_idx);
    schedule_part.buf[slash_idx] = '\0';
    str_trim(&schedule_part);

    // 分割脚本路径
    strcpy(script_path.buf, slash_ptr + 1);
    str_trim(&script_path);
    if (strlen(script_path.buf) == 0)
        return false;

    ScheduledTask task;
    memset(&task, 0, sizeof(task));
    strcpy(task.script_path, script_path.buf);

    // 分支1：纯数字，每N分钟
    if (str_all_digit(&schedule_part)) {
        int minutes = str_to_int(&schedule_part);
        if (minutes > 0) {
            strcpy(task.schedule_type, "minutely");
            task.minutes_interval = minutes;
            task.next_execution = calculate_minutely_time(minutes, time(NULL));
            task_queue_push(task_queue, &task);
            return true;
        }
    }

    // 查找空格，判断周计划格式
    char* space_ptr = strchr(schedule_part.buf, ' ');
    if (space_ptr) {
        String first_part, second_part;
        size_t sp_idx = space_ptr - schedule_part.buf;
        strncpy(first_part.buf, schedule_part.buf, sp_idx);
        first_part.buf[sp_idx] = '\0';
        str_trim(&first_part);

        strcpy(second_part.buf, space_ptr + 1);
        str_trim(&second_part);

        if (str_all_digit(&first_part)) {
            int weekday = str_to_int(&first_part);
            if (weekday >= 1 && weekday <= 7) {
                int h, m;
                if (parse_time_string(&second_part, &h, &m)) {
                    strcpy(task.schedule_type, "weekly");
                    task.weekday = weekday;
                    task.hour = h;
                    task.minute = m;
                    task.next_execution = calculate_weekly_time(weekday, h, m, time(NULL));
                    task_queue_push(task_queue, &task);
                    return true;
                }
            }
        }
    }

    // 分支3：每日定时 HH:MM
    int hour, minute;
    if (!parse_time_string(&schedule_part, &hour, &minute))
        return false;

    strcpy(task.schedule_type, "daily");
    task.hour = hour;
    task.minute = minute;
    task.next_execution = calculate_daily_time(hour, minute, time(NULL));
    task_queue_push(task_queue, &task);
    return true;
}

// ------------------- 工具函数实现 -------------------
// 去除首尾空格
static void str_trim(String* s) {
    char* start = s->buf;
    while (*start && isspace((unsigned char)*start)) start++;
    char* end = start + strlen(start) - 1;
    while (end >= start && isspace((unsigned char)*end)) end--;
    *(end + 1) = '\0';
    memmove(s->buf, start, strlen(start)+1);
}

// 判断字符串是否全数字
static bool str_all_digit(const String* s) {
    for (int i = 0; s->buf[i]; i++) {
        if (!isdigit((unsigned char)s->buf[i]))
            return false;
    }
    return strlen(s->buf) > 0;
}

// 字符串转int
static int str_to_int(const String* s) {
    return atoi(s->buf);
}
```

## 其他定时方案
在Linux系统当中，运用C++进行网络服务或者是后台程序的开发之际，定时器是一个相当关键重要的组件。不管是心跳检测，还是超时控制，亦或是任务调度以及缓存刷新，全都离不开稳定可靠的定时器机制。Linux给出了多种实现定时器的方式，每一种方式都有着它自身适用的场景以及性能特点，这些都是需要依据具体需求来进行选择的。 Linux下C++定时器有哪些实现方式 最为传统型的方式乃是运用alarm系统调用以及信号处理，然而此种方式精准度较低并且信号处理繁杂，于现代应用里已然较少被采用了。select/poll/epoll等I/O多路复用机制能够结合超时参数达成简易的定时功能，这属于网络编程当中常见的做法。 还有一种途径是运用POSIX timer系列函数，像timer_create、timer_settime这类，它们能够给出更高的精准度以及更具灵活性的管控，除此之外红旗linux下载，Linux独有的timerfd机制把定时器抽象成文件描述符，能够天衣无缝地融入基于事件驱动的程序里，对于那些需要复杂调度的情形，还能够借助时间轮等数据结构在用户空间自行达成定时器管理。 select定时器的工作原理是什么 选取函数自身是一个输入/输出多路复用接口，它的最后那一个参数超时时间能够用以达成定时功能。在调用选取时，要是没有文件描述符准备好，函数会处于阻塞状态直至超时时间来临。虽说选取的精度通常是毫秒级别，然而对于大多数网络应用来讲已然充足。 要留意的是，select的timeout参数于返回之际会被变更为剩余的时间，所以每一次调用之前都得再度进行设置。另外，select能够监控的文件描述符的数量是有限的（一般为1024），在高并发的场合之下有可能会变成瓶颈。尽管select定时器的实现较为简易，然而在对性能有着较高要求的场景当中或许并非是最佳的选择。
